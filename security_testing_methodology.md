# 安全测试方法论

## 核心方法论

### 后台日志搜索
对有意义的用户输入（如文件名、特殊 payload、长字符串）操作后，根据上下文判断，在后台日志中搜索原始输入，结合上下文分析记录情况。

### 后台文件搜索
涉及文件操作后，在后台日志搜索文件名，确认文件实际落盘目录及记录情况。

### 时间关联分析
- 要结合时间分析，避免分析非当前操作的日志。
- 注意：前后端时间可能不完全一致，需留出合理的时间窗口。

### 全量日志监控
如果能确定相关操作对应的后台日志文件路径，可以尝试全量监控这些日志文件的新增内容，因为有时候一些关键信息不一定包含前端输入信息。

### 全面操作监控
所有用户操作都可以尝试监控后台是否涉及：
- 文件操作
- 命令执行
- 网络请求

### 关键目录与文件分析
后台涉及相关的关键目录和文件可以尝试打开分析，甚至可以执行脚本和命令进行深入探查。

### 综合分析
后台获取的关键信息可以结合代码综合分析。
