# 安全测试方法论

## 1. 测试流程

采用五阶段框架，"核心方法论"中的观察/验证动作贯穿其中。

### 信息收集
- 梳理资产、接口、角色、依赖组件、已知 CVE。
- 明确测试范围与边界，避免越权或影响无关系统。

### 测绘建模
- 识别输入点与信任边界，标注用户可控字段。
- 推断关键数据流，以及可能的落盘/子进程/外联行为。

### 动态测试
- 针对输入点注入带观察性的 payload（可辨识的特征串）。
- 同步开启后台日志、文件、命令、网络维度的监控。

### 验证定性
- 对可疑点做最小化 PoC，排除环境与副作用干扰。
- 判断可复现性、触发条件、影响面与依赖前提。

### 记录上报
- 标注置信度（见第 4 节）。
- 写明复现前提、已完成与缺失的验证步骤，而非简单结论。

## 2. 核心方法论

### 后台日志搜索
对有意义的用户输入（如文件名、特殊 payload、长字符串）操作后，根据上下文判断，在后台日志中搜索原始输入，结合上下文分析记录情况。

### 后台文件搜索
涉及文件操作后，在后台搜索文件名，确认文件实际落盘目录及记录情况。

### 时间关联分析
- 结合时间窗口分析，避免把非当前操作的日志当成证据。
- 前后端时间可能不完全一致，搜索时留出合理窗口。

### 全量日志监控
如果能确定相关操作对应的后台日志文件路径，可以全量监控这些日志文件的新增内容，因为有时候关键信息不一定包含前端输入。

### 全面操作监控
所有用户操作都尝试三问：**是否落盘？是否拉起子进程？是否外联？** 对应监控：
- 文件操作
- 命令执行
- 网络请求

### 关键目录与文件分析
后台涉及的关键目录和文件可以直接打开分析，必要时执行脚本或命令做深入探查。

### 综合分析
后台获取的关键信息结合代码综合分析，形成完整的调用链与判断依据。

## 3. 漏洞类别观察点

针对常见漏洞类型，给出在后台应重点关注的信号。

- **命令注入**：是否拉起子进程、shell 解释器参数、异常命令行日志。
- **文件上传 / 任意写**：落盘路径、文件名是否保留原始输入、文件权限位。
- **路径穿越**：日志中出现 `..`、或解析后的绝对路径是否越出预期目录。
- **SSRF / 外联**：异常目的 IP/域名、内网地址、非预期的 DNS 解析请求。
- **反序列化 / 模板注入**：类加载异常、反射调用、模板引擎错误栈。
- **SQL 注入**：SQL 错误日志、慢查询、拼接式 SQL 痕迹。
- **越权**：日志中用户 ID 与资源归属不一致的记录。
- **信息泄露**：错误栈、调试接口、备份文件、目录列举命中。

## 4. 漏洞置信度要求

漏洞报告必须标注置信度，分为 **高 / 中 / 低** 三档。仅通过代码分析（白盒）得出的结论一般置信度不高，需结合黑盒或灰盒验证以提升置信度。

### 高置信度
- 已通过黑盒或灰盒验证，能稳定复现。
- 具备完整的 PoC（请求/响应、命令执行输出、落盘文件、日志记录等实际证据）。
- 漏洞影响与利用路径明确，无歧义。

### 中置信度
- 代码分析已较为充分，逻辑链完整，但缺少端到端的运行时验证。
- 或黑灰盒验证部分成立（例如触发了异常/日志，但未拿到完整利用证据）。
- 需要补充动态验证后才能升为高置信度。

### 低置信度
- 仅基于代码分析，未做任何黑灰盒验证。
- 依赖较多假设条件（特定配置、特定权限、特定环境）且未确认。
- 作为线索登记，需要进一步验证才能定性。

### 验证原则
- 所有仅凭代码分析得出的漏洞，默认视为低置信度，必须补充黑灰盒验证。
- 黑灰盒验证应尽量在真实/仿真的环境中进行，并结合后台日志、文件、命令、网络等多维度证据交叉印证。
- 置信度不足时，应明确说明缺失的验证步骤，而非直接上报为确认漏洞。

## 5. 常见陷阱与反干扰

- **时间不一致**：前后端或多机时钟偏差，搜日志要留窗口，必要时以服务端时间为准。
- **日志脱敏**：敏感字段（密码/手机号/身份证等）在日志中可能被 `****`，直接搜原始 payload 会漏判；应搜截断前后缀、上下游字段或关联 ID。
- **异步/批处理延迟**：日志不一定即时落盘，存在 flush、队列、定时任务延迟，分析时要等一个周期再下结论。
- **缓存命中**：重复请求可能走缓存不到后台，换参数或加扰动位再测。
- **WAF / 前置网关**：payload 在前置被改写或拒绝，后台看不到，要对比网关日志与业务日志。
- **多副本 / 多节点**：请求入口节点与实际处理节点未必相同，监控要覆盖所有副本，或先定位到实际处理节点。
- **编码差异**：URL / Unicode / Base64 等在不同层解码后形态不同，搜日志时同时搜原文与解码形态。
